Data Intel & Model Architecture

This system is built upon a dataset of over 45,000 delivery records, specifically optimized to predict the Target Variable (Output): Time_taken (min).

MODEL OUTPUT (TARGET)

Delivery Latency

Continuous numeric value representing total duration from order pickup to customer drop-off.

FEATURE TAXONOMY (INPUTS)

Personnel Metrics

Age, ratings, and experience profile of the delivery courier.

Geo-Spatial Data

Haver-sine calculated distance between restaurant and destination.

Network Density

Real-time traffic flow (Low, Medium, High, Jam) at time of order.

Climate Inputs

Atmospheric conditions like Fog, Stormy, or Sunny impact on mobility.

The underlying model is a Stacking Regressor that combines multiple base learners to minimize RMSE. Data is version-controlled via DVC and the training lifecycle is managed through MLflow.
